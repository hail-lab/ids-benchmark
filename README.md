# A Comprehensive Benchmark of Machine Learning Models for Network Intrusion Detection: Multi-Dataset Evaluation With SHAP Explainability

## Overview

This repository implements a unified benchmark of eight ML/DL models for network intrusion detection, evaluated across three standard IDS datasets (CICIDS2017, ToN-IoT, UNSW-NB15) under identical leakage-controlled preprocessing and five-fold stratified cross-validation for both binary and multi-class tasks (48 model--dataset--task combinations).

**Paper:** *A Comprehensive Benchmark of Machine Learning Models for Network Intrusion Detection: Multi-Dataset Evaluation With SHAP Explainability* — submitted to IEEE Access.

**Authors:** S. Aljaloud and A. Alshammari, University of Ha'il, Saudi Arabia.

---

## Results Summary

| Finding | Key Result |
|---|---|
| **Data Leakage Inflation** | Up to 68% F1 inflation when identifier features retained (ToN-IoT); dose--response across 3 datasets |
| **LightGBM Failure Mode** | F1 collapses from 0.998 (binary) to 0.197 (multi-class) on CICIDS2017; validated via 3 alternative configs |
| **SHAP Feature Divergence** | Entirely disjoint top features: TCP window sizes (CICIDS2017), Zeek metadata (ToN-IoT), TTL features (UNSW-NB15) |
| **Feature-Selection Ablation** | 80% feature reduction with < 0.4% F1 loss across all datasets |
| **Best Overall Model** | XGBoost — top multi-class robustness; LightGBM — fastest binary training |
| **Friedman/Nemenyi** | Tree ensembles significantly outperform DL models across all datasets |

## Reproduction

### 1. Install dependencies

```
pip install -r requirements.txt
```

### 2. Download datasets

```
python src/data_collection.py
```

This prints manual download instructions for each dataset. Place the downloaded files in `data/raw/`:

| Dataset | Source | Rows | Classes | Year |
|---|---|---|---|---|
| **CICIDS2017** | [UNB CIC](https://www.unb.ca/cic/datasets/ids-2017.html) | ~2.8M | 15 | 2017 |
| **ToN-IoT** | [UNSW](https://research.unsw.edu.au/projects/toniot-datasets) | ~460K | 10 | 2020 |
| **UNSW-NB15** | [UNSW](https://research.unsw.edu.au/projects/unsw-nb15-dataset) | ~2.5M | 10 | 2015 |

### 3. Preprocess all datasets

```
python src/preprocessing.py
```

### 4. Train all models (8 models x 3 datasets x 2 tasks = 48 combinations)

```
python src/model.py
```

### 5. Post-training analysis

```
python src/post_training.py
python src/ablation.py
python src/friedman_cd.py
python src/confusion_matrices.py
```

### 6. Generate figures and SHAP analysis

```
python src/evaluation.py --shap
```

### 7. Leakage experiment and LightGBM validation

```
python src/leakage_comparison.py
python src/lgbm_validation.py
python src/lgbm_validation_ova.py
```

### 8. Generate LaTeX tables

```
python src/gen_latex_tables.py
```

All outputs (CSV tables, PDF figures) are saved to `outputs/tables/` and `outputs/figures/`.

### Selective runs

```
python src/preprocessing.py --dataset cicids2017
python src/model.py --dataset cicids2017 --model XGBoost
python src/ablation.py --cross-dataset
python src/ablation.py --fs-ablation
```

## Analysis Code (`src/`)

### Data Collection and Preprocessing
- `data_collection.py` — prints download URLs and instructions for all three datasets
- `preprocessing.py` — unified cleaning pipeline: drops leakage columns (IPs, ports, flow IDs), encodes labels, outputs parquet
- `utils.py` — shared paths, logging configuration, and constants

### Model Training
- `model.py` — trains all 8 models (XGBoost, Random Forest, LightGBM, SVM, k-NN, MLP, 1D-CNN, BiLSTM) with 5-fold stratified CV; saves results to CSV and model checkpoints

### Evaluation and Analysis
- `evaluation.py` — generates performance figures (grouped bars, heatmaps, scatter plots) and SHAP beeswarm plots
- `post_training.py` — computes per-class F1, binary-to-multi degradation, and Wilcoxon signed-rank tests
- `ablation.py` — feature-selection ablation (MI + RF hybrid) and cross-dataset transfer analysis
- `friedman_cd.py` — Friedman test with Nemenyi post-hoc and Critical Difference diagrams
- `confusion_matrices.py` — normalised confusion matrix heatmaps for multi-class results

### Leakage and Robustness
- `leakage_comparison.py` — compares clean vs. leaky (identifier-retaining) results; generates leakage impact bar charts
- `lgbm_validation.py` — validates LightGBM multi-class failure through alternative configurations
- `lgbm_validation_ova.py` — LightGBM one-vs-all validation experiment

### Utilities
- `gen_latex_tables.py` — auto-generates LaTeX table code from result CSVs

- `per_class_f1.csv` — per-class F1 scores for all models
